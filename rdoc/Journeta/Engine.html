Journeta::Engine

The primary fascade of the entire Journeta library, which composite a number of objects running asynchronously to the primary application Thread. Use of this fascade requires a minimal amount of lifecycle management on your part to start and stop the engine at appropriate times. (Usually only at application startup and shutdown, respectively.)

Constants

DEFAULT_PEER_PORT

Incoming direct peer TCP connections will use this port.

DEFAULT_PRESENCE_PORT

On UDP port on which we send/receive peer presence messages.

DEFAULT_PRESENCE_NETWORK

Addresses 224.0.0.0 through 239.255.255.255 are reserved for multicast messages.

DEFAULT_PRESENCE_PERIOD

The wait time, in seconds, between rebroadcasts of peer presence.

Attributes

uuid[R]

A supposedly universally unique id for this instance.

groups[R]

An array of peer network names. Ex: [‘Preston Lee Ventures Test’, ‘quick_chat_app’] An empty array indicates implicit membership in all discovered groups.

presence_broadcaster[R]

Continuously sends out “i’m here” presence messages to the local network

presence_listener[R]

continuously listens for “i’m here” presence messages from other peers

presence_port[R]

The UDP port for event broadcast messages.

presence_address[R]

The UDP network address used for broadcast messages.

presence_period[R]

The amount of time between presence broadcasts.

peer_listener[R]

Constantly listens for incoming peer sessions

peer_handler[R]

Application logic which processes session data.

peer_registered_handler[R]

Application logic which gets notified when a peer comes online.

peer_updated_handler[R]

Application logic which gets notified when a peer comes online.

peer_unregistered_handler[R]

Application logic which gets notified when a peer goes offline.

peer_port[R]

The TCP port used to receive direct peer messages.

peer_registry[R]

Authoritative peer availability database.
