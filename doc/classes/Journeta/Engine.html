Class Journeta::Engine
In: lib/journeta/journeta_engine.rb
Parent: Object

The primary fascade of the entire Journeta library, which composite a number of objects running asynchronously to the primary application Thread. Use of this fascade requires a minimal amount of lifecycle management on your part to start and stop the engine at appropriate times. (Usually only at application startup and shutdown, respectively.)

Public Instance methods

known_groups()
known_peers(all_groups = false)

Returns metadata on all known peers in a hash, keyed by the uuid of each. A record corresponding to this peer is not included.

register_peer(peer)

Adds (or updates) the given PeerConnection. If a peer of the same UUID is found, the existing record will be updated and given instance PeerConnection#stop‘d. This prevents pending outbound data from being accidentally dropped.

send_to_known_peers(payload)

Sends the given object to all peers in one of the groups associated with this instance. The object will be marshalled via YAML, so anything the YAML serializer misses won‘t make it to the other side(s). The return value is undefined.

send_to_peer(peer_uuid, payload)

Send the given object to the peer of the given UUID, if available. The return value is undefined.

start()

Starts sub-comonents which have their own life-cycle requirements. The registry itself does not have a dedication thread, and thus does not need to be started.

stop()
unregister_peer(peer)

Forcefully unregisters the given PeerConnection, though this is of limited use since the PresenceListener will eventually automatically re-register the peers UUID if it‘s still online.
